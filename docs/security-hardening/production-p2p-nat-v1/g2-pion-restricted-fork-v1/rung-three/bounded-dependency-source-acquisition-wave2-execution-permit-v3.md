# G2 Pion dependency wave-two v3 execution permit

This reader contract accompanies the machine-validated v3 permit. The permit
authorizes one fresh, sequential intake of the exact 15 tuple pairs and 30
`.mod` then `.zip` requests from `https://proxy.golang.org`.

Wave2 v2 remains consumed and immutable. Its permit, claim, failure receipt,
tool bytes, four attempted and completed responses, three validated resources,
and empty terminal namespace are predecessor evidence. v3 does not retry,
resume, delete, backfill, or reuse any v2 response or staging byte.

The historical 200:1 compression value is telemetry only. A ZIP is never
rejected merely for exceeding it. The runner records exact integer numerator
and denominator bytes for the first maximum-ratio entry and an advisory
exceeded flag. It uses no division, floating point, entry name, or body in that
telemetry.

All absolute and structural limits remain mandatory: 16 MiB per ZIP, 1 MiB per
`.mod`, 64 MiB aggregate response, 20,000 entries per archive, 300,000 entries
aggregate, 128 MiB single-file and per-archive uncompressed, 1 GiB aggregate
uncompressed, 8 MiB central directory, bounded paths, verified CRC and EOF,
central/local header agreement, no ZIP64, encryption, symlink, special file,
duplicate, casefold collision, or extraction, exact raw SHA-256, module ZIP H1,
`.mod` H1, and optional embedded `go.mod` parity.

ZIP inspection, prepublication inventory, and postpublication verification are
inside the whole-wave hard deadline. Any first prepublication failure consumes
the v3 claim, removes staging, records a bounded failure receipt, publishes no
partial final set, and forbids retry. Any postpublication uncertainty is
reported as terminal uncertainty and also forbids retry.

Success remains incomplete until the separate offline readback checker reopens
all 30 files, independently reproduces every identity, structure, H1, bound,
and compression telemetry result, and publishes its receipt then manifest.

No repository login, owner proof, private key, signature, token, password,
credential, or user action is required. Product endpoint authentication is a
separate runtime invariant.
