# Protocol Schema

`protocol.schema.json` defines the v1 message envelope and active v0.1 payload shapes for the client-to-runtime protocol.

`docs/protocol.md` remains the source of behavioral semantics, transport sequencing, and future extension rules. The schema is intentionally limited to active runtime messages. Basic `memory.list`, `memory.upsert`, `memory.delete`, bounded exact `memory.duplicate_suggestions.list`, bounded model-dependent `memory.semantic_duplicate_suggestions.list` pair review, bounded complete-link `memory.semantic_duplicate_clusters.list` cluster review, and capability-gated `research.brief.create` plus `research.notebooks.list` operations are active. Research briefs reuse approved trusted-source grants and the existing chat stream, history, cancellation, and source-attribution contracts; notebook summaries expose no grant, source-text, path, URL, backend, token, or embedding metadata. `research.notebooks.list` uses strict mutually exclusive initial, cursor-only continuation, legacy response, and `research.notebooks.authoritative_sync.v1` response branches, with legacy pages capped at 100 and authoritative pages capped at 200 against a snapshot count of at most 10,000. `hello.client_capabilities` remains unique and nonblank and is bounded to 64 entries. The fixture and validators provide no-device/no-network wire-contract evidence only; they do not establish live runtime snapshot, physical-device, provider, or network behavior. The threshold calibration foundation remains an offline or explicitly invoked loopback evaluation tool and adds no message or capability; automatic merge, external research/network access, reflection, and roadmap namespaces such as skills, MCP, and web search stay out of the active enum until implemented.
